## ⚙️ Operational Protocols (Mandatory)
*   **Automatic Tracking:** Every significant action, modification, or task execution is recorded in `live_tracker.md` immediately, without user prompting.
*   **Mandatory Timestamps:** Every entry in the live tracker and all project reports MUST include a precise Date & Time stamp (UTC) as and when they are updated.
*   **Friday Reviews:** A comprehensive project status review is conducted every Friday to sync milestones and plan the following week.
*   **Clean Handover:** All summaries and handover documents must be derived directly from the `live_tracker.md` to ensure zero-loss context for future sessions.
*   **Versioned History:** Original "Version 1.0" data is preserved as an immutable archive; new work starts at Version 1.1.
*   **Gatekeeper Protocol:** No live updates (GitHub/Netlify pushes) or version increments are permitted without a full preview and explicit permission being granted in the active session.
*   **Delta Verification:** Whenever asked if the tracker or memory is updated, the response must start by checking the delta from the last state to the current state to ensure perfect continuity.

## 🏗️ Maintenance Architecture (Safe Editing)
*   **Data-Isolate & Decouple:** All content for Academy, Ecosystem, and Library is stored in centralized manifest files (`src/data/*.ts`). Modifications to content must happen in these data files to prevent breaking the UI logic.
*   **Atomic Expansion:** New features must use pre-existing Atomic Components (`src/components/`) to maintain brand consistency.
*   **Route Integrity:** Page additions/removals are managed within the `(site)` Route Group to isolate the main site from cinematic/teaser engines.

## 📊 Reporting & Versioning Protocol
*   **Snapshot-First:** Before any significant change (v2.0, v3.0, etc.), a snapshot of the current stable state must be preserved (e.g., via Git tags or architectural freezes).
*   **Version Increment:** Every discrete enhancement or change request triggers a new version number (v1.1, v1.2, etc.). The "Original v1.0" code is treated as an immutable baseline.
*   **Mandatory Checkpoints:** Every task completion must culminate in a "Checkpoint Report" covering Branding, Build Integrity, Mobile Responsiveness, and Tracker Sync.
*   **Real-Time Sync:** All changes are reflected in `live_tracker.md` immediately upon execution.
*   **Atomic Rollbacks:** Every deployment on Netlify is preserved as a unique atomic state. In the event of a v1.1+ failure, the system allows for an instant 1-click rollback to the v1.0 baseline.
